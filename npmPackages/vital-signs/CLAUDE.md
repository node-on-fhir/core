# CLAUDE.md — @node-on-fhir/vital-signs

Migrated from Atmosphere `vital-signs` (2026-06-13, MIT). HL7 FHIR Vital Signs IG — panels (BP, HR, RR, SpO2, temp, weight, height, BMI, head circ.) + recording/display. 3 routes (`/vital-signs`, `/vital-signs/:patientId`, `/vital-signs/:patientId/:observationId`, all → VitalSignsPageWrapper). **NOT externally gated** — the `clinical:vital-signs` api.use is an onTest self-ref.

- **Entries:** Atmosphere had `mainModule('index.jsx','client')` + `mainModule('lib/index.js','server')`. client.js builds routes/sidebar from workflow.json, exports DynamicRoutes/SidebarElements/SidebarWorkflows, and `export * from './lib/index.js'` (the shared surface: VitalSigns collection handle, VitalSignsSchemas, VitalSignsValueSets, Extensions, Utilities, package meta). server.js = `export * from './lib/index.js'` (the server mainModule). Added the `Meteor` import index.jsx relied on as an Atmosphere global.
- **Collection collision fix:** `lib/index.js` binds `VitalSigns` to the host app's `Observations` collection; when `Meteor.Collections.Observations` isn't populated yet (NPM/Rspack load order) it fell through to `new Mongo.Collection('Observations')` → collision. Fixed with `{ _suppressSameNameError: true }` (runs on both client Minimongo + server).
- **Dead server tree:** `server/` (index.js + methods/{create,update,delete}VitalSign.js + publications/vitalSigns.js) was **never wired into package.js** (no `server/` reference; server mainModule is lib/index.js) → kept for reference but NOT imported, faithful to the original. (Wiring it is a possible future enhancement.)
- **Charts / deps:** the package shipped both a recharts and a nivo chart impl (defaulting to recharts). recharts could NOT be added as an npm dep without an ERESOLVE that rewrites the whole workspace lockfile (its react-dom peer collides with a react-dom@19 in the tree), so VitalSignsChartWrapper was changed to use the **nivo** impl only (`@nivo/line`, already present, peer). `charts/recharts/` kept for reference but unimported (so Rspack does not need recharts). simpl-schema peer. MUI v5. `guide/` (HL7 cimi-vital-signs submodule) + `tests/` + `configs/` skipped. `favorite`→`Favorite`. Not in `.meteor/packages`. Monorepo-tracked (guide submodule) → fresh git init.
